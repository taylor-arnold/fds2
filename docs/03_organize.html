3  Organizing Data I

3.1 Setup

We will begin this and every subsequent chapter by listing the core modules and datasets that must be loaded before getting started. Datasets and sources that require new functions will be loaded within the body of the text.

import polars as pl
from plotnine import ggplot, aes
from polars import col as c

import funs

country = pl.read_csv("data/countries.csv")

3.2 Introduction

This chapter covers the core methods for organizing a dataset: reordering its rows, choosing which columns to keep, and filtering down to the rows you care about. We will also look at how to create new columns from existing ones, rename columns, and remove duplicate rows. Every method in this chapter works on a single DataFrame at a time, one row or one column at a time, with no notion of groups. Later chapters build on these same methods once we start comparing categories of data against each other.

All of the operations below come from the Polars library, the same one we used to read our data. Each takes a DataFrame as input and returns a modified DataFrame as output, which means the methods can be chained together in a single pipeline. Individually, these methods are simple. Chained together, they let you turn a raw dataset into exactly the shape you need for an analysis.

To keep a sorted dataset for later use, store it in a named variable:

country_saved = (
    country
    .sort(c.region, c.pop)
)

After running this code, country_saved holds the reordered dataset, while country still holds the original. The same pattern applies to the results of any method described in this chapter.

We can filter categorical variables with equality comparisons too. To keep the page from getting too cluttered, we will stop printing the result of every example from here on.

(
    country
    .filter(c.region == "Europe")
)

This keeps every row where region == "Europe". Note the double-equals operator (==) for testing equality; a single equals sign (=) is reserved for assignment.

We can combine multiple conditions with the & (and) and | (or) operators. Wrap each condition in parentheses when doing so:

(
    country
    .filter((c.region == "Europe") & (c.hdi > 0.9))
)

Common mathematical transformations are available by adding a . after the column name, followed by a function such as sqrt() or log10().

The following example takes the square root and the sine of population and saves each as a new column. Neither transformation is meaningful here; they are only meant to show the technique.

(
    country
    .with_columns(
      pop_sqrt = c.pop.sqrt(),
      pop_sin = c.pop.sin()
    )
)

Note the formatting: when a column definition is long, or when defining several columns at once, put each one on its own line, indented by four spaces.

NoteAlias method

Polars includes a method called .alias that can be used within .with_columns to create or overwrite a column. It was the original standard technique but has largely been replaced by the equals-sign syntax used above, which most people find more intuitive. It’s worth recognizing because you will still see it often in the Polars documentation.

3.8 Pipes

Chaining methods together is a powerful pattern. It keeps code readable, avoids naming a pile of intermediate variables, and lets Polars optimize the whole sequence of operations internally. One limitation of method chaining is that it only gives you access to methods built into Polars. The .pipe method gets around this by letting you call an arbitrary function with the DataFrame as its first argument. .pipe takes the function to apply as its first argument, and passes any additional arguments straight through to that function.

As a simple example, we can apply the helper function print_rows to a DataFrame to control how many rows are shown, instead of the default 10. Here we select 4, which shows the first two rows and the last two.

3.9 Further Functions

One more function is worth knowing before moving on: pl.when, which creates a column that depends on a condition elsewhere in the data. The example below builds a column equal to 1 for countries in the Northern Hemisphere and 0 for those in the Southern Hemisphere.

(
    country
    .with_columns(
        is_north = pl.when(c.lat >= 0).then(1).otherwise(0)
    )
)

pl.when can chain additional .when().then() pairs before the final .otherwise(), which makes it a compact way to build a column out of several conditions at once.

3.10 R’s dplyr and Python Pandas

If you’re familiar with dplyr in R or pandas in Python, the code in this chapter should feel familiar. Function names differ, and each library has its own quirks, but the underlying task is the same: compose a set of methods to transform data into the shape a given analysis needs.

Coming from dplyr, the main adjustments are prefixing column names with c., using methods instead of pipes (the same idea, different syntax), and a few renamed functions—sort in place of arrange, for instance.

Coming from Pandas involves a few more changes, most of which simplify the code. There is no index column to reset after every operation, no aggregation functions wrapped in tuples, and no eval or query methods buried behind an extra layer of quoted strings.

As with Pandas and dplyr, Polars also provides functions for pivots and table joins, which we cover in Chapter 9. The arguments differ slightly across the three libraries, but the underlying transformations are much the same.
